Request Fingerprinting

Every /check request is gated behind a verified browser fingerprint session. Automated scripts, curl, and headless browsers are blocked at multiple independent layers:

① Browser Signal Collection
15 signals collected in the browser — canvas pixel hash, WebGL renderer, audio context fingerprint, screen geometry, timezone, fonts, plugins, and hardware concurrency. These cannot be accurately faked without running a real browser engine.
② Proof of Work (PoW)
The browser must find a solution where SHA-256(nonce + solution) starts with 2 leading zero bytes (~65,536 hashes on average, ~65ms in a browser Web Worker). A Python script using hashlib needs ~130ms per token — plus PoW is combined with the per-fingerprint rate limit (max 10 sessions/hour), making bulk token farming impractical.
③ Bot-Signal Gate
navigator.webdriver = true (set by Selenium and Playwright) is hard-rejected server-side. outerWidth = 0 && outerHeight = 0 (default headless Chrome) is also rejected. These checks happen before any token is issued.
④ HMAC-Signed Nonce + Time Gate
The server issues a cryptographically signed nonce valid for 60 s. The token request must arrive within 55 s of challenge issuance — pre-computed or cached responses are rejected. The nonce is consumed atomically (single-use) to prevent replay.
⑤ Double httpOnly Cookie Binding
Two independent httpOnly cookies are set: fp_token (HMAC-signed, contains session ID) and fp_sid (opaque session ID stored in Redis). Both must be present and mutually consistent. JavaScript cannot read httpOnly cookies — they are invisible to document.cookie and devtools console. An attacker needs to steal both cookies independently.
⑥ Per-Request Single-Use Token
Before every /check call, the browser fetches a one-time X-Request-Token from /fingerprint/request-token. The server consumes it atomically (Redis GETDEL) and rejects any reuse. This defeats "Copy as cURL" from DevTools: the copied command contains a token already spent by the browser's own request. Even if an attacker has the session cookies, they cannot replay a captured /check — the token is gone.
⑦ Long-Lived Session + Server Revocation
Sessions last for a configurable TTL (default 24 h) — no per-request re-fingerprinting. The session is stored in Redis under fp:session:<id>. Operators can immediately revoke any session by deleting that key — useful for abuse response without waiting for expiry.
⑧ Per-Fingerprint Rate Limit
Max 10 new sessions per unique fingerprint hash per hour — no IP address is stored (privacy-compliant). A bot that rotates browsers to generate new fingerprints must solve PoW for each, capped at 10 sessions/hour per device profile.
⑨ Origin Enforcement
The challenge and token endpoints check the Origin header (with Referer fallback for same-origin GETs) against ALLOWED_ORIGINS. Requests from curl, Python scripts, or other origins with no matching header are rejected with 403 before any processing occurs.
